# Скилл: Модель AnyLogic "СРесурсами1"

## Общее описание
Модель симулирует рабочий коллектив. Метод: дискретно-событийное моделирование (СМО). Язык: Java. Программа: AnyLogic 8.

## Основная цепочка (Main)
Source(задания1) → Queue(queue1) → Seize → Delay → Release → Sink

## Три ветки модели
| Ветка | Файл | Назначение |
|---|---|---|
| Оригинальный парадокс | СРесурсами1ПервоначальныйСПарадоксом | Архив, для публикаций леваки |
| 2.1 Без парадокса | 2.1.СРесурсами2ВчерашнийБезПарадокса | Реалистичная, с `(1.0-здоровье)*0.003` |
| 2.2 Рабочая | 2.2СРесурсамиСегодняшнийПарадоксальный | Психотипы + MBTI + соционика + типЗадания |

## Класс Работник
**Переменные:**
- `здоровье`, `продуктивность` — начальное triangular(0.3, 1.3, 0.8)
- `зарплата = 700 * продуктивность` (переменная, не параметр!)
- `больничНачало`, `больничКонец`, `суммаБольничных`, `суммаЗарплат`
- `когдаВзятНаРаботу = -uniform(0, 365)` — отрицательное для разброса стажа
- `больничныйЧасов` — вспомогательная
- `накопленнаяУсталость` — растёт во время работы, снижается в перерывы/болезнь
- `коэффициентПрезентеизма = 1.0` — снижает продуктивность при выгорании
- `психотип` — int (1=шизоид, 2=эпилептоид, 3=гипертим, 4=истероид, 5=нормотим)
- `психиатрическийПсихотип` — ПсихиатрическиеПсихотипы
- `mbti` — ТипМБТИ
- `соционика` — ТипСоционика
- `экстраверсия`, `интуиция`, `логика`, `рациональность`, `доминантность` — double (0-1)

**Переменная `болеет` удалена** — race condition. Используется `inState(заболел)`.

**Диаграмма состояний (ВзятНаРаботу):**
- `работает` → `заболел` по сообщению `"заболей"`
- `заболел` → `работает` по сообщению `"выздоровел"`
- `работает` → `перегорает` — таймаут 10000 дней (заглушка)
- `перегорает` → finalState

**При входе в заболел:** `больничНачало = time();`
**При выходе из заболел:** `больничКонец = time();`

**Событие СуммированиеЗарплат (каждый час):**
```java
if (!inState(заболел) && main.расписаниеПерерывов.getValue() == false) {
    суммаЗарплат += зарплата;
    накопленнаяУсталость += 0.003;
    // В ветке 2.1 (без парадокса) дополнительно:
    // накопленнаяУсталость += Math.max(0, (1.0 - здоровье) * 0.003);
} else {
    накопленнаяУсталость = Math.max(накопленнаяУсталость - 0.0005, 0);
}
```

## Настройки блоков Main

### ResourcePool
- Тип: Движущийся, количество: `количествоРаботников`
- Перерывы: включены, расписание: `расписаниеПерерывов`, вытеснение: ✓
- Аварии/ремонты:
  - До первой: `Math.max(uniform(0, 365*unit.здоровье), 1)` дней
  - Между: `Math.max(triangularAV(365*unit.здоровье*0.95, 0.1*unit.здоровье), 1)` дней
  - Ремонт: `Math.min(Math.max(triangularAV(2/Math.max(unit.здоровье,0.01), 0.5/Math.max(unit.здоровье,0.01)), 0.01), 30.0/7.0)` недель
- **При поломке:** `unit.больничНачало = time(); send("заболей", unit);`
- **При починке:**
```java
send("выздоровел", unit);
unit.больничКонец = time();
unit.суммаБольничных += (unit.больничКонец - unit.больничНачало) * 24 *
    unit.зарплата * коэффециэнтДляБольничных;
```

### Seize
- Правило вытеснения: Ожидать оригинал ресурса, авто приостановка/возобновление: ✓
- При подготовке ресурса:
```java
Работник работник = (Работник)unit;
agent.назначенноеВремяДэлэйПоДанномуСотруднику =
    0.105 * agent.сложность / (работник.продуктивность * работник.здоровье);

boolean совпадение =
    (работник.психотип == 1 && agent.типЗадания == ТипЗадания.аналитическое) ||
    (работник.психотип == 2 && agent.типЗадания == ТипЗадания.регламентное)  ||
    (работник.психотип == 3 && agent.типЗадания == ТипЗадания.управленческое)||
    (работник.психотип == 4 && agent.типЗадания == ТипЗадания.творческое);

if (совпадение) {
    agent.назначенноеВремяДэлэйПоДанномуСотруднику *= 0.7;
    работник.накопленнаяУсталость -= 0.05;
    работник.накопленнаяУсталость = Math.max(работник.накопленнаяУсталость, 0);
} else {
    agent.назначенноеВремяДэлэйПоДанномуСотруднику *= 1.3;
    работник.накопленнаяУсталость += 0.1;
}
```
Коэффициент 0.105: без здравоохранения ~33% заданий, с здравоохранением ~100%.
Нормотим (психотип=5) никогда не совпадает.

### Delay
- Время: `agent.назначенноеВремяДэлэйПоДанномуСотруднику`

### Release
```java
agent.выручкаЗаЗадание = 3 * agent.назначенноеВремяДэлэйПоДанномуСотруднику * ((Работник)unit).зарплата;
main.суммаВыручки += agent.выручкаЗаЗадание;
```

### Source → "При выходе"
```java
double rv = uniform(0, 1);
if (rv < доляАналитических)
    agent.типЗадания = ТипЗадания.аналитическое;
else if (rv < доляАналитических + доляРегламентных)
    agent.типЗадания = ТипЗадания.регламентное;
else if (rv < доляАналитических + доляРегламентных + доляУправленческих)
    agent.типЗадания = ТипЗадания.управленческое;
else
    agent.типЗадания = ТипЗадания.творческое;
```

## Исправленные баги
1. Здоровье без нижней границы → краш. `Math.max(..., 0.05)`
2. Событие увольнение occuresOnce → cyclic
3. Race condition с `болеет` → удалена, используется `inState(заболел)`
4. Единицы суммаБольничных → множитель 24
5. Таймаут перегорает 320 дней → 10000 дней
6. когдаВзятНаРаботу = 0 → `-uniform(0, 365)`
7. задержкаНайма стартовала сама → начальное время 99999
8. restartTo принимает абсолютное время → `restartTo(time() + uniform(...))`
9. remove работники (пробел) → `remove_работники`
10. Код психотипов попадал в суммаЗарплатИБольничных → восстановлен правильный код

## Единицы измерения
- Время модели: **дни**
- Зарплата: рублей **в час** (базовая 500)
- `больничКонец - больничНачало` → дни → умножать на 24

## ПЛАН РАБОТ

### 🔴 Срочно

**1. Переменная `навык`** *(первостепенное)*
10% "железных" работников: `if (uniform(0,1) < 0.1) навык = triangular(0.8,1.5,1.2); else навык = triangular(0.1,0.8,0.4);`
Влияет на: скорость деградации здоровья (`/навык`), накопление усталости (`/навык`), скорость заданий в Seize.

**2. Переход `работает → перегорает` по условию** *(первостепенное)*
Заменить таймаут 10000 дней на `накопленнаяУсталость >= 4`.
При входе: `коэффициентПрезентеизма = 0.5`.
В Seize добавить `* работник.коэффициентПрезентеизма` в знаменатель.

**3. Выгоревший остаётся балластом** *(первостепенное)*
В `перегорает` работник получает зарплату, работает с `коэффициентПрезентеизма = 0.5`.
Выход: uniform(300,500) дней → `работает` (восстановился) или увольнение если здоровье < 0.3.

**4. Выход из `перегорает`** *(первостепенное)*
Связан с п.3. После восстановления: `накопленнаяУсталость = 0`, `коэффициентПрезентеизма = 1.0`.

---

### 🟡 Малосрочные

**5. Выгорание от перенапряжения (сила воли)** *(первостепенное)*
В Seize: `работник.накопленнаяУсталость += Math.max(0, (agent.сложность / работник.продуктивность - 30) * 0.01);`

**6. Трудоголик поднимает планку для всех** *(первостепенное)*
Переменная `базоваяСложность` в Main растёт на 5-10% каждые полгода если команда перевыполняет план. Ударяет по среднячкам.

**7. Отпуска снижают выгорание** *(первостепенное)*
Раз в год uniform(14,28) дней: работник недоступен, `накопленнаяУсталость -= 0.05/час`.

**8. Параметр стабильности / Акционер** *(вторичное)*
`индексСтабильности` = СКО выручки за 90 дней. Резкие колебания → штраф к бюджету.

**9. времяБезРаботника** *(первостепенное)*
Пока нового не наняли — очередь растёт, выручка падает.

**10. частотаБольничных** *(вторичное)*
Внешний множитель на timeBetweenFailures (сезон, эпидемия).

**11. Зависимость зарплаты от навыка** *(вторичное)*
`зарплата = базоваяЗарплата * продуктивность * (1 + навык * 0.3)`

### 🔵 Далёкая перспектива

**12. Руководители с тёмной тетрадой** *(первостепенное)*
Поднимаются из популяции работников с экстремальными значениями 5 осей:
- Нарциссизм: экстраверсия > 0.9, логика < 0.3
- Макиавеллизм: логика > 0.9, рациональность > 0.85
- Психопатия: экстраверсия > 0.8, логика > 0.8, рациональность < 0.2
- Садизм: доминантность > 0.85, логика > 0.7
Влияние: смещают реальный профиль задач, ускоряют выгорание команды.

**13. Отбор через рынок труда (эволюция психотипов)**
Новый работник приходит из популяции со смещением психотипов в зависимости от профиля компании. Через N поколений найма доминируют совместимые психотипы.

**14. Прогрессия акцентуации в расстройство**
При длительном совпадении психотипа с задачей акцентуация усиливается — шизоид на аналитике через 5 лет ближе к клиническому аутизму.

**15. Автоматизация меняет спрос на психотипы**
`доляРегламентных` постепенно → 0, сложность оставшихся задач растёт.

**16. База данных, ESG-метрики, ROI, NPV** *(вторичное)*
- `коэффициентАбсентеизма`, `индексВыгорания`, `eNPS`
- `ROI = (экономияБольничных + приростПродуктивности + экономияТекучести - стоимостьПрограммы) / стоимостьПрограммы × 100%`
- `NPV = суммаВыручки(с_здрав) - суммаВыручки(без_здрав) - стоимостьПрограммы`
