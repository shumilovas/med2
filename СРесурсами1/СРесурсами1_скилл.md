# Скилл: Модель AnyLogic "СРесурсами1"

## Общее описание
Модель симулирует рабочий коллектив. Метод: дискретно-событийное моделирование (СМО). Язык: Java. Программа: AnyLogic 8.8.

## Основная цепочка (Main)
Source(задания1) → Queue(queue1) → Seize → Delay → Release → Sink

## Структура проекта
- **Main** — основная логика
- **Задание** — класс заявки
- **Работник** — класс ресурса
- **Simulation: Main** — эксперимент со слайдерами
- **База данных** — пока не используется
- **Ресурсы** — 3D модели, не важны

## Класс Работник

**Переменные:**
- `здоровье` — начальное `triangular(0.3, 1.3, 0.8)`
- `продуктивность` — начальное `triangular(0.3, 1.3, 0.8)`
- `зарплата = 700 * продуктивность` (переменная, не параметр!)
- `больничНачало`, `больничКонец` — моменты времени
- `суммаБольничных` — накопленные расходы на больничные
- `суммаЗарплат` — накопленная зарплата работника
- `когдаВзятНаРаботу` — момент найма (отрицательный при инициализации)
- `больничныйЧасов` — вспомогательная
- `накопленнаяУсталость` — растёт в рабочее время (кап: 101), снижается в перерывы/болезнь
- `коэффициентПрезентеизма = 1.0` — снижает продуктивность при выгорании (пока не подключён в формуле)
- `коэффициентУтомления = 1.0` — множитель скорости накопления усталости, зависит от здоровья
- `психотип` — int (1=шизоид, 2=эпилептоид, 3=гипертим, 4=истероид, 5=нормотим)
- `психиатрическийПсихотип` — ПсихиатрическиеПсихотипы
- `mbti` — ТипМБТИ
- `соционика` — ТипСоционика
- `экстраверсия`, `интуиция`, `логика`, `рациональность`, `доминантность` — double (0–1)
- `участвуетВПрограмме` — boolean, `uniform() < 0.5` при инициализации
- `уровеньАктивности = 0` — накапливается +0.01/час у участников программы
- `премияЗаАктивность = 0` — рассчитывается в расчетПремииЗаФизАкт
- `absenteeismProbability = 0.2` — **задел, пока не подключён**

**Переменная `болеет` удалена** — race condition. Используется `inState(заболел)`.

---

### Диаграмма состояний (ВзятНаРаботу)
- `работает` → `заболел` по сообщению `"заболей"`
- `заболел` → `работает` по сообщению `"выздоровел"`
- `работает` → `перегорает` — таймаут **10000 дней** (заглушка, замена на условие в плане)
- `перегорает` → finalState

При входе в `заболел`: `больничНачало = time();`
При выходе из `заболел`: `больничКонец = time();`

### Событие СуммированиеЗарплатИУсталости (каждый час)
```java
if (!inState(заболел) && main.расписаниеПерерывов.getValue() == false
        && накопленнаяУсталость < 101) {
    суммаЗарплат += зарплата;
    накопленнаяУсталость += 0.003 * коэффициентУтомления;
} else {
    накопленнаяУсталость = Math.max(накопленнаяУсталость - 0.0005, 0);
}
```
Один механизм накопления (парадоксальная ветка). Второй механизм `+= Math.max(0, (1.0-здоровье)*0.003)` **удалён**. Кап 101 — усталость не уходит в бесконечность.

### Событие пересчетПроизводительностиПриВыгорании (раз в месяц, первое через 2 месяца)
```java
if (накопленнаяУсталость > 25) {
```

Третий множитель — доля дней на больничном. ⚠️ Продуктивность умножается каждый месяц, нужна нижняя граница.

## Настройки блоков Main

### ResourcePool
- Тип: Движущийся, количество: `количествоРаботников`, новый ресурс: Работник
- Перерывы: `расписаниеПерерывов`, может вытеснять: ✓
- Время до первой аварии: `Math.max(uniform(0, 365*unit.здоровье), 1)` дней
- Время между авариями: `Math.max(triangularAV(365*unit.здоровье*0.95, 0.1*unit.здоровье), 1)` дней
- Время ремонта: `Math.min(Math.max(triangularAV(2/Math.max(unit.здоровье,0.01), 0.5/Math.max(unit.здоровье,0.01)), 0.01), 30.0/7.0)` недель

При поломке:
```java
unit.больничНачало = time();
send("заболей", unit);
```
При починке:
```java
send("выздоровел", unit);
unit.больничКонец = time();
unit.суммаБольничных += (unit.больничКонец - unit.больничНачало) * 24 *
    unit.зарплата * коэффециэнтДляБольничных;
```

### Seize
- Правило вытеснения: Ожидать оригинал ресурса. Авто приостановка: ✓
- При подготовке ресурса:
```java
Работник работник = (Работник)unit;

// Базовое время выполнения
agent.назначенноеВремяДэлэйПоДанномуСотруднику =
    0.105 * agent.сложность / (работник.продуктивность * работник.здоровье);

// Совпадение психотипа с типом задания
boolean совпадение =
    (работник.психотип == 1 && agent.типЗадания == ТипЗадания.аналитическое) ||
    (работник.психотип == 2 && agent.типЗадания == ТипЗадания.регламентное)  ||
    (работник.психотип == 3 && agent.типЗадания == ТипЗадания.управленческое)||
    (работник.психотип == 4 && agent.типЗадания == ТипЗадания.творческое);

if (совпадение) {
    agent.назначенноеВремяДэлэйПоДанномуСотруднику *= 0.7;
    работник.накопленнаяУсталость -= 0.05;
    работник.накопленнаяУсталость = Math.max(работник.накопленнаяУсталость, 0);
} else {
    agent.назначенноеВремяДэлэйПоДанномуСотруднику *= 1.3;
    работник.накопленнаяУсталость += 0.1;
}
```
Коэффициент 0.105: без здравоохранения ~33% заданий, с ним ~100%.
Нормотим (психотип=5) никогда не совпадает — всегда ×1.3 и усталость растёт.

### Delay
- Время: `agent.назначенноеВремяДэлэйПоДанномуСотруднику`

### Release
```java
agent.выручкаЗаЗадание = 3 * agent.назначенноеВремяДэлэйПоДанномуСотруднику * ((Работник)unit).зарплата;
main.суммаВыручки += agent.выручкаЗаЗадание;
```

### НазначаемИзначальноеЗдоровьеИПсихотипы (1 раз через 1 час)
```java
for (int i = 0; i < работники.size(); i++) {
    работники.get(i).здоровье = triangular(0.3, 1.3, 0.8);
    работники.get(i).продуктивность = triangular(0.3, 1.3, 0.8);
    работники.get(i).когдаВзятНаРаботу = -uniform(0, 365);

    // Психотип: 20/20/20/15/25%
    double rand = uniform(0, 1);
    if      (rand < 0.20) работники.get(i).психотип = 1;
    else if (rand < 0.40) работники.get(i).психотип = 2;
    else if (rand < 0.60) работники.get(i).психотип = 3;
    else if (rand < 0.75) работники.get(i).психотип = 4;
    else                  работники.get(i).психотип = 5;

    // MBTI-параметры по психотипу через triangular(min, max, mode)
    // Шизоид: низкая экстраверсия, высокая интуиция и логика
    // Эпилептоид: высокая логика и рациональность
    // Гипертим: высокая экстраверсия и доминантность
    // Истероид: высокая экстраверсия, низкая логика
    // Нормотим: все параметры средние (0.3–0.7)

    // Из булевых e/n/t/j выводится mbti и соционика (16 веток if-else)

    // Программа физактивности
    работники.get(i).уровеньАктивности = 0;
    работники.get(i).премияЗаАктивность = 0;
    работники.get(i).участвуетВПрограмме = uniform() < 0.5;
    работники.get(i).рассчитатьПремиюЗаФизическуюАктивность();
}
```

## Исправленные баги
1. Здоровье без нижней границы → краш. `Math.max(..., 0.05)`
2. Событие увольнение occuresOnce → cyclic
3. Race condition с `болеет` → удалена, используется `inState(заболел)`
4. Единицы суммаБольничных: дни × руб/час → добавлен множитель 24
5. Таймаут перегорает 320 дней → 10000 дней (заглушка)
6. когдаВзятНаРаботу = 0 у всех → `-uniform(0, 365)`
7. задержкаНайма стартовала сама → начальное время 99999
8. restartTo принимает абсолютное время → `restartTo(time() + uniform(...))`

## Известные баги (не исправлены)
1. **обновитьСостояние**: пороги здоровья `85` и `70` вместо `0.85` и `0.70` — условие `коэффициентУтомления` никогда не срабатывает
2. **пересчетПроизводительностиПриВыгорании**: продуктивность умножается каждый месяц без нижней границы — может уйти к нулю
3. **absenteeismProbability = 0.2** — переменная добавлена, но нигде не используется

## ПЛАН РАБОТ

### 🔴 Срочно

**1. Исправить баг в обновитьСостояние** *(блокирующее)*
Заменить `85` → `0.85` и `70` → `0.70`.

**2. Добавить нижнюю границу продуктивности** *(блокирующее)*
В `пересчетПроизводительностиПриВыгорании`:
```java
продуктивность = Math.max(продуктивность * ..., 0.1);
```

**3. Подключить absenteeismProbability** *(первостепенное)*
Вероятность отсутствия без больничного. Механика: в `СуммированиеЗарплатИУсталости` — если `uniform() < absenteeismProbability` и усталость высокая → не начислять зарплату, не накапливать усталость (работник "не пришёл"). Отличается от болезни: не вытесняет задание из Seize.

**4. Переход `работает → перегорает` по условию** *(первостепенное)*
Заменить таймаут 10000 дней на условие `накопленнаяУсталость >= 4` (или выше, подобрать).
При входе в `перегорает`: `коэффициентПрезентеизма = 0.5`.
Добавить `* работник.коэффициентПрезентеизма` в формулу Seize.

**5. Выход из `перегорает`** *(первостепенное)*
Таймаут `uniform(300, 500)` дней → если `здоровье > 0.3`: вернуть в `работает`, сбросить усталость. Иначе → finalState.

---

### 🟡 Малосрочные

**6. Переменная `навык` в Работнике**
10% "железных" работников: `triangular(0.8, 1.5, 1.2)`, остальные `triangular(0.1, 0.8, 0.4)`.
