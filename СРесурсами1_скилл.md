# Скилл: Модель AnyLogic "СРесурсами1"

## Общее описание
Модель симулирует рабочий коллектив. Метод: дискретно-событийное моделирование (СМО). Язык: Java. Программа: AnyLogic 8.

## Основная цепочка (Main)
Source(задания1) → Queue(queue1) → Seize → Delay → Release → Sink

## Структура проекта
- **Main** — основная логика
- **Задание** — класс заявки
- **Работник** — класс ресурса
- **Simulation: Main** — эксперимент со слайдерами
- **База данных** — пока не используется
- **Ресурсы** — 3D модели, не важны

## Класс Работник
**Параметры:**
- (нет — зарплата перенесена в переменные)

**Переменные:**
- `здоровье` — начальное triangular(0.3, 1.3, 0.8)
- `продуктивность` — начальное triangular(0.3, 1.3, 0.8)
- `зарплата = 700 * продуктивность` (переменная, не параметр!)
- `болеет` — boolean
- `больничНачало`, `больничКонец` — моменты времени
- `суммаБольничных` — накопленные расходы на больничные
- `суммаЗарплат` — накопленная зарплата
- `когдаВзятНаРаботу` — момент найма

**Диаграмма состояний (ВзятНаРаботу):**
- `работает` → `заболел` по сообщению "заболей"
- `заболел` → `работает` по сообщению "выздоровел"
- `работает` → `перегорает` по таймауту 320 дней (заглушка, больше длины симуляции)
- `перегорает` → finalState

**При входе в заболел:**
```java
больничНачало = time();
```
**При выходе из заболел:**
```java
больничКонец = time();
суммаБольничных += (больничКонец - больничНачало) * зарплата * main.коэффициентДляБольничных;
```

**Событие СуммированиеЗарплат (каждый час):**
```java
if (болеет == false && main.расписаниеПерерывов.getValue() == false)
    суммаЗарплат += зарплата;
```

## Настройки блоков Main

### ResourcePool
- Тип: Движущийся
- Количество ресурсов: `количествоРаботников`
- Новый ресурс: Работник
- Перерывы: включены, расписание: `расписаниеПерерывов`
- Может вытеснять (перерыв): ✓
- Аварии/ремонты: включены
  - Время до первой аварии: `uniform(0, 365*unit.здоровье)` дней
  - Время между авариями: `triangularAV(365*unit.здоровье*0.95, 0.1*unit.здоровье)` дней
  - Время ремонта: `triangularAV(2/unit.здоровье, 0.5/unit.здоровье)` недели
  - Тип ремонта: Задержка
- При поломке: `unit.болеет = true; send("заболей", unit);`
- При починке:
```java
unit.болеет = false;
send("выздоровел", unit);
```
- При создании нового ресурса: здоровье и продуктивность НЕ назначаются здесь — назначаются через событие НазначаемИзначальноеЗдоровье

### Seize
- Правило вытеснения задач: Ожидать оригинал ресурса
- Авто приостановка/возобновление: ✓
- При подготовке ресурса:
```java
Работник работник = new Работник();
работник = (Работник)unit;
agent.назначенноеВремяДэлэйПоДанномуСотруднику = 2*agent.сложность/(работник.продуктивность*работник.здоровье);
```

### Delay
- Время: `agent.назначенноеВремяДэлэйПоДанномуСотруднику`

## События Main

### НазначаемИзначальноеЗдоровье
- Срабатывает 1 раз через 1 час
```java
for (int i = 0; i < работники.size(); i++) {
    работники.get(i).здоровье = triangular(0.3, 1.3, 0.8);
    работники.get(i).продуктивность = triangular(0.3, 1.3, 0.8);
}
```

### ЕстественноеСнижениеЗдоровья
- Циклическое, каждые 400 часов
```java
if (!РаботаетЛиСистемаЗдравосохранения) {
    for (int i=0; i < работники.size(); i++)
        работники.get(i).здоровье -= 0.05;
}
```

### ПрограммаЗдравосохранения
- Циклическое, каждые 800 часов, первое срабатывание в 0
```java
if (РаботаетЛиСистемаЗдравосохранения == true) {
    for (int i=0; i < работники.size(); i++)
        работники.get(i).здоровье = Math.min(работники.get(i).здоровье + 0.2, 2.3);
    if (длительностьБолезни > 5) длительностьБолезни -= 5;
}
```

### суммаЗарплатИБольничныхСобытие
- Циклическое, каждый час, первое срабатывание через 10 часов
```java
суммаЗарплат = 0;
суммаБольничных = 0;
for(int i=0; i < работники.size(); i++) {
    суммаЗарплат += работники.get(i).суммаЗарплат;
    суммаБольничных += работники.get(i).суммаБольничных;
}
```

### увольнение
- Циклическое, каждые 30 дней
```java
for (int i = работники.size()-1; i >= 0; i--) {
    Работник р = работники.get(i);
    if ((time() - р.когдаВзятНаРаботу > 120) &&
        (р.здоровье * р.продуктивность > 1.5)) {
        double коэффициент = uniform(1.1, 1.5);
        double новаяЗарплата = р.зарплата * коэффициент;
        double текущаяСумма = 0;
        for (int j = 0; j < работники.size(); j++)
            текущаяСумма += работники.get(j).зарплата;
        if (текущаяСумма - р.зарплата + новаяЗарплата <= бюджетНаЗарплаты)
            р.зарплата = новаяЗарплата;
        else
            remove_работники(р);
    }
}
```

## Что не реализовано / заглушки
1. **Коэффициент 2 в формуле** — даёт ~109 часов на задание, нужно уменьшить
2. **База данных** — не используется
3. **Выручка** — не считается, бюджетНаЗарплаты пока фиксированный
4. **перегорает** — заглушка, таймаут 320 дней (больше длины симуляции)
5. **времяБезРаботника** — запланировано для моделирования поиска замены
